  <name>Task 2: Apply responsive design fixes based on audit</name>
  <files>frontend3/App.tsx, frontend3/components/Sidebar.tsx, frontend3/tailwind.config.js</files>
  <action>
    Based on audit findings, apply fixes:

    **Sidebar.tsx:**
    - Add mobile breakpoint behavior (hidden on small screens, hamburger menu trigger)
    - Ensure sidebar collapses to icons-only on tablet if space-constrained
    - Add slide-out drawer behavior for mobile (off-canvas menu)

    **App.tsx:**
    - Add hamburger menu button for mobile (visible only on small screens)
    - Ensure top header adapts (hide search bar on mobile, adjust button visibility)
    - Verify main content area takes full width when sidebar is hidden

    **General fixes:**
    - Replace fixed widths with responsive alternatives: `w-full md:w-80`
    - Add overflow handling: `overflow-x-hidden` on body/container
    - Ensure font sizes scale appropriately: `text-xl md:text-2xl`
    - Fix padding/margins for mobile: `px-4 md:px-8`

    If hamburger menu state management needed:
    - Add useState to App.tsx for mobile menu open/close
    - Wire hamburger button to toggle menu
    - Add overlay/backdrop for mobile menu
  </action>

<task type="checkpoint:human-verify" gate="blocking">
  <what-built>Responsive design fixes applied across frontend components</what-built>
  <how-to-verify>
    1. Start dev server: `cd frontend3 && npm run dev`
    2. Open browser DevTools (F12)
    3. Enable Responsive Design Mode (device toolbar)

    **Test Desktop (1280px+):**
    - Set viewport to 1280x720
    - Verify: Full sidebar visible, main content takes remaining space
    - Verify: Search bar visible in header
    - Verify: "New Document" button visible in header
    - Check: No horizontal scroll at any height

    **Test Tablet (768px - 1279px):**
    - Set viewport to 768x1024
    - Verify: Sidebar either collapses to icons or has hamburger toggle
    - Verify: Search bar may be hidden (acceptable)
    - Verify: All content accessible without horizontal scroll
    - Check: Layout adapts gracefully

    **Test Mobile (<768px):**
    - Set viewport to 375x667 (iPhone SE)
    - Verify: Hamburger menu button visible in header
    - Click hamburger: Verify sidebar opens as overlay/drawer
    - Click outside or X: Verify sidebar closes
    - Verify: Content stacks vertically (no side-by-side layouts)
    - Check: All buttons are tap-friendly (min 44px height)
    - Check: Text is readable without zooming

    **Test Extreme Mobile (320px):**
    - Set viewport to 320x568
    - Verify: No horizontal scroll
    - Verify: All content accessible
    - Check: Layout doesn't break

    For each viewport:
    - Check all main views: Dashboard, Files, Library, Editor
    - Verify no content overflow
    - Verify interactive elements are clickable

    Document any issues found with viewport size and specific element.
  </how-to-verify>
  <resume-signal>Type "approved" if responsive design works at all sizes, or describe issues found</resume-signal>
