## Integration Issues

### Issue #1: AnalysisView Missing ProjectContext Integration

**Severity:** Medium
**Phase:** 11 (View Integration) / 13 (Realtime Features)
**File:** `/home/zemul/Programming/research/frontend3/pages/AnalysisView.tsx`
**Line:** 12

**Details:**
- AnalysisView has a TODO comment: `// TODO: Get from route/context`
- Uses hardcoded `projectId = 'default-project'` instead of `currentProjectId` from context
- All other views (FilesView, EditorView, MemoryView) properly use `useProjectContext()`

**Impact:**
- Analysis executions are not associated with the actual current project
- Findings from analysis are saved to wrong project (or non-existent 'default-project')
- Inconsistent with Phase 16 memory API integration pattern

**Recommended Fix:**
1. Add `import { useProjectContext } from '../context/ProjectContext';`
2. Replace `const [projectId] = useState('default-project');` with `const { currentProjectId } = useProjectContext();`
3. Remove TODO comment
