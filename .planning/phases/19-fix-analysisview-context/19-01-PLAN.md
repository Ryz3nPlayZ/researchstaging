Fix AnalysisView ProjectContext Integration

This plan closes a MEDIUM priority integration gap from the v1.1 milestone audit. AnalysisView currently uses a hardcoded `projectId = 'default-project'` instead of retrieving the current project ID from ProjectContext. This causes analysis executions to be associated with the wrong (or non-existent) project.

Purpose: Ensure data analysis results are associated with the correct project
Output: Updated AnalysisView.tsx that uses currentProjectId from context

The fix is straightforward:
1. Add import for useProjectContext from '../lib/context'
2. Replace `const [projectId] = useState('default-project');` with `const { currentProjectId } = useProjectContext();`
3. Remove the TODO comment on line 12
4. Update handleExecute to use currentProjectId (with null check)
</objective>
