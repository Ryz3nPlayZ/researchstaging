# Plan 02-02: File Explorer Frontend Component

**Phase:** 2 - File & Project Management
**Plan:** 2 of 4
**Type:** Feature
**Wave:** 1 (Can start after 02-01)
**Depends on:** 02-01 (needs robust backend API)
**Files modified:** `frontend/src/components/files/` (new directory)
**Autonomous:** Yes (independent frontend work)
**Must haves:**
1. Tree view of project files and folders
2. Drag-and-drop file upload zone
3. Create/rename/delete folder operations
4. File list view with metadata (type, size, date)
5. Context menu for file operations (download, delete)

## Overview

Build the primary file management UI component that displays project files in a tree structure with drag-and-drop upload. This component will be integrated into the Navigator panel and provides the main interaction point for users to organize their research files.

## Tasks

### Task 1: Create FileExplorer component with tree view

**File:** `frontend/src/components/files/FileExplorer.jsx` (new)

**Changes:**
- Create React component that mounts in Navigator panel
- Fetch file tree from `/api/files/projects/{project_id}/files/tree`
- Render tree structure with folders (expandable) and files (leaf nodes)
- Use shadcn/ui components (TreeView or custom recursive rendering)
- Show file icons by type (PDF icon, code icon, etc.)
- Display file metadata on hover (size, type, modified date)

**Success criteria:**
- Tree renders all folders and files correctly
- Folders expand/collapse on click
- Files show appropriate icons by type
- Empty state shown when project has no files

### Task 2: Implement drag-and-drop upload zone

**File:** `frontend/src/components/files/FileExplorer.jsx`

**Changes:**
- Add drop zone component at top of file tree
- Handle `dragover`, `dragleave`, `drop` events
- On drop: upload files to selected folder or root
- Show upload progress for each file
- Prevent non-file drops (text, links)
- Validate file types on client side before upload

**Success criteria:**
- Files upload when dropped on component
- Visual feedback during drag (highlight drop zone)
- Upload progress shown for each file
- Multiple files upload simultaneously
- Invalid files rejected before upload

### Task 3: Add folder operations (create, rename, delete)

**File:** `frontend/src/components/files/FileExplorer.jsx`

**Changes:**
- Add "New Folder" button with dialog for folder name
- Add context menu on right-click for folders (rename, delete)
- Implement rename functionality (inline edit or dialog)
- Implement delete with confirmation dialog
- Update tree view immediately after operations
- Handle errors gracefully (folder not empty, permission denied)

**Success criteria:**
- New folder created in selected location
- Folders renamed with validation (no empty names)
- Folders deleted with confirmation
- Folder tree updates in real-time
- Error messages shown for failed operations

### Task 4: Add file operations (download, delete, move)

**File:** `frontend/src/components/files/FileExplorer.jsx`

**Changes:**
- Add context menu for files (download, delete, move)
- Download: trigger browser download from `/api/files/{file_id}/download`
- Delete: show confirmation, then call DELETE `/api/files/{file_id}`
- Move: drag file to folder (or use context menu → move to folder)
- Add "Copy path" action for file reference
- Update file tree after operations

**Success criteria:**
- Files download when download clicked
- Files deleted with confirmation
- Files moved by dragging to folder
- File paths copied to clipboard
- Tree updates immediately after operations

### Task 5: Create file API client hooks

**File:** `frontend/src/lib/api.js` (extend existing)

**Changes:**
- Add `uploadFile(projectId, file, folderId)` function
- Add `createFolder(projectId, name, parentId)` function
- Add `deleteFile(fileId)` function
- Add `deleteFolder(folderId)` function
- Add `renameFolder(folderId, name)` function
- Add `moveFile(fileId, folderId)` function
- Handle errors and return structured responses
- Integrate with existing axios instance (auth headers)

**Success criteria:**
- All file operations available as API calls
- Errors caught and transformed to user-friendly messages
- Auth tokens automatically included
- File uploads use FormData correctly
- Loading states provided for async operations

## Definition of Done

- [ ] Drag-and-drop upload works for all file types
- [ ] Folder operations (create, rename, delete) functional

- [ ] Accessible (keyboard navigation, ARIA labels)

## Testing Strategy

1. Create folder → verify appears in tree
2. Upload file via drag-drop → verify uploaded and shows in tree
3. Rename folder → verify name updates in tree
4. Delete file → verify removed from tree
5. Download file → verify browser downloads
6. Drag file to folder → verify moves to folder
7. Try uploading unsupported file → verify client-side validation
8. Test error cases (network failure, server error)
