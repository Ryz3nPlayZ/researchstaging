## Cloudflare R2 Configuration (Recommended)

R2 is recommended over AWS S3 for this use case because:

1. **Zero egress fees** - Research datasets can be large (hundreds of MB to GB)
2. **S3-compatible API** - Drop-in replacement with boto3
3. **Lower cost** - No data transfer fees for downloads

Setup steps:
1. Create Cloudflare account
2. Enable R2 in dashboard
3. Create bucket (e.g., `research-workspace-files`)
4. Create API token with R2 permissions
5. Configure environment variables:

```bash
STORAGE_BACKEND=r2
S3_ENDPOINT_URL=https://<account-id>.r2.cloudflarestorage.com
S3_BUCKET_NAME=research-workspace-files
S3_ACCESS_KEY_ID=<R2 token access key>
S3_SECRET_ACCESS_KEY=<R2 token secret>
```

## AWS S3 Configuration (Alternative)

Use S3 if you already have AWS infrastructure:

```bash
STORAGE_BACKEND=s3
S3_BUCKET_NAME=research-workspace-files
S3_REGION=us-east-1
S3_ACCESS_KEY_ID=<AWS access key>
S3_SECRET_ACCESS_KEY=<AWS secret key>
```
