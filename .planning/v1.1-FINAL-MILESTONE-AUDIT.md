## Critical Integration Issues

### Issue 1: Memory API Route Mismatch (HIGH)

**Problem:**
```typescript
// Frontend: lib/api.ts calls:
memoryApi.search(query, limit)
// → GET /memory/search?q=...&limit=...

// Backend expects:
GET /memory/projects/{project_id}/claims/search
```

**Impact:** Memory view search completely broken — returns 404 errors

**Files:** `frontend3/lib/api.ts`, `frontend3/pages/MemoryView.tsx`

**Fix:** Update `memoryApi` to include `projectId` parameter and correct routes

───────────────────────────────────────────────────────────────

### Issue 2: WebSocket Connection Port (MEDIUM)

**Problem:**
```typescript
// App.tsx line 25:
connect(currentProjectId, window.location.origin);
// → ws://localhost:3000/ws/{projectId}
// Backend is at: ws://localhost:8000/ws/{projectId}
```

**Impact:** Real-time features (task updates, collaboration) will not work

**Files:** `frontend3/lib/websocket.ts`, `frontend3/App.tsx`, `vite.config.ts`

**Fix:** Configure Vite proxy with `ws: true` or use `VITE_API_URL` for WebSocket
