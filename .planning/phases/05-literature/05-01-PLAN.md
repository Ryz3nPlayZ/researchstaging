Create new file backend/literature_api.py with REST endpoints:

1. GET /api/literature/search
   - Query params: query (string), limit (int, default 20)
   - Calls LiteratureService.search_all()
   - Enhances results with Unpaywall lookup for DOI-based papers
   - Sorts results: open-access PDFs first, then by citation count
   - Returns: List of paper objects with metadata and PDF URLs

2. GET /api/literature/papers/{paper_id}
   - Get detailed paper metadata by ID
   - Includes full abstract, all authors, references, citation count

3. POST /api/literature/papers/{paper_id}/download
   - Downloads PDF from pdf_url
   - Saves to backend/storage/pdfs/{paper_id}.pdf
   - Returns file path and metadata

Error handling:
- 400 for invalid query parameters
- 404 if paper not found
- 500 for API rate limiting or download failures (log error)

- Log all searches for monitoring
