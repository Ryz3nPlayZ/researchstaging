  <name>Task 1: Create Pandoc-based export service</name>
  <files>backend/export_service.py</files>
  <action>
    Create ExportService class following established service layer patterns:

    1. **TipTap to Pandoc conversion:**
       - Convert TipTap JSON to Markdown (pandoc input format)
       - Extract document structure: headings (h1-h6), paragraphs, lists, bold/italic, blockquotes
       - Handle citations: convert citation nodes to placeholder text or pandoc citations

    2. **Pandoc execution:**
       - Use subprocess.run() to execute pandoc CLI
       - PDF output: `pandoc input.md -o output.pdf --pdf-engine=xelatex`
       - DOCX output: `pandoc input.md -o output.docx`
       - Include timeout (30 seconds) and error handling

    3. **Temporary file management:**
       - Use /tmp for intermediate files (input.md, output.pdf, output.docx)
       - Clean up temp files after conversion
       - Return file bytes or file path as configured

    4. **Error handling:**
       - PandocNotFoundError: Pandoc not installed (clear error message)
       - ConversionError: Pandoc conversion failed (include stderr)
       - TimeoutError: Conversion took too long

    5. **Reuse existing patterns:**
       - Follow document_service.py async pattern
       - Use Pydantic models for request/response validation
       - Include comprehensive logging
  </action>

<task type="auto">
  <name>Task 2: Create export API endpoints</name>
  <files>backend/api/export.py</files>
  <action>
    Create export router following FastAPI router pattern from document_api.py:

    1. **Router setup:**
       - Create APIRouter at /api/documents/{document_id}/export
       - Include tags: ["Export"]

    2. **Export endpoints:**
       - GET /api/documents/{document_id}/export/pdf
         - Query params: project_id (for ownership validation)
         - Returns: FileResponse with content-disposition header
         - Filename: {document_title}.pdf

       - GET /api/documents/{document_id}/export/docx
         - Query params: project_id (for ownership validation)
         - Returns: FileResponse with content-disposition header
         - Filename: {document_title}.docx

    3. **Ownership validation:**
       - Verify document belongs to project_id
       - Verify project belongs to authenticated user (via token)
       - Return 403 if validation fails

    4. **Integration with ExportService:**
       - Load document by document_id
       - Parse TipTap JSON from document.content
       - Call ExportService.export_to_pdf() or export_to_docx()
       - Stream file response to client

    5. **Error responses:**
       - 404: Document not found
       - 403: Ownership validation failed
       - 500: Export failed (Pandoc error, include detail message)
       - 503: Pandoc not installed (clear message: "Export service unavailable")

    6. **Register router:**
       - Add to backend/server.py: app.include_router(export_router)
  </action>
  <verify>
    - Export endpoints registered and accessible via /docs
    - GET /api/documents/{id}/export/pdf returns PDF file with correct headers
    - GET /api/documents/{id}/export/docx returns DOCX file with correct headers
    - Ownership validation enforced (403 for unauthorized access)
  </verify>
  <done>
    Export API endpoints provide PDF and DOCX downloads with proper validation, error handling, and file response headers.
  </done>
</task>
