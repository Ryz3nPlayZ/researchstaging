# Research summary from Phase 16 research
Backend routes are project-scoped: `/memory/projects/{project_id}/*`
Frontend API client uses incorrect routes (search endpoint doesn't exist, claims/findings missing project_id)
ProjectContext provides currentProjectId hook (used by FilesView, EditorView)
MemoryView doesn't use ProjectContext - needs to import and pass currentProjectId

# Backend route reference (memory_api.py)
- GET /memory/projects/{project_id}/claims/search?q={query}&limit={limit} -> List[ClaimResponse]
- GET /memory/projects/{project_id}/claims -> List[ClaimResponse]
- GET /memory/projects/{project_id}/findings -> List[FindingResponse]
- GET /memory/projects/{project_id}/claims/{claim_id}/relationships -> List[ClaimRelationshipResponse]
