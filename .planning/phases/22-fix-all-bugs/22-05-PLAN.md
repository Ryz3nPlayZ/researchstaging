# Regression Results Summary

**Test Date:** 2026-02-08
**Pass Rate:** 14.3% (3/21 verified working)
**Status:** INCOMPLETE - Gap closure required

## Issues to Fix (7 total)

### P0 Critical Blockers (5)

1. **Literature Search - "no claims found" error**
   - Symptom: Literature search returns "no claims found" instead of papers
   - Severity: P0 (blocks core research workflow)
   - Root Cause Hypotheses:
     - Semantic Scholar API rate-limited or down
     - Search query not matching database
     - Backend parsing logic has edge cases
     - Frontend not displaying results correctly
   - Investigation: Check backend response, verify API connectivity, examine parsing logic

2. **AI Chat - "sorry i encountered an error"**
   - Symptom: Chat returns error message when sending
   - Severity: P0 (blocks AI assistance features)
   - Root Cause Hypotheses:
     - LLM API key missing or invalid
     - Backend route unhandled
     - WebSocket connection failing
     - Error handling swallowing actual error
   - Investigation: Check backend logs, verify LLM API credentials, test endpoint directly

3. **Bibliography Button - does nothing**
   - Symptom: Clicking bibliography button has no effect
   - Severity: P0 (blocks citation workflow)
   - Root Cause Hypotheses:
     - Click handler not attached
     - Backend endpoint missing
     - No papers/citations to bibliography
     - UI disabled but not visually indicated
   - Investigation: Check EditorView button handler, verify API endpoint

4. **Project Navigation - only works on "all projects" tile**
   - Symptom: Recent projects tile doesn't navigate, only all projects tile
   - Severity: P0 (inconsistent navigation)
   - Root Cause: Click handler only attached to all projects tile
   - Fix: Apply same handler to recent projects tile

5. **Create Project - navigates to document view**
   - Symptom: After creating project, navigates to document instead of workspace
   - Severity: P0 (architectural mismatch)
   - User Feedback: "a project should be a workspace/repository (documents, chats, tasks, graphs, analyses), not just a document"
   - Root Cause: Project creation flow assumes document-first view
   - Fix: Navigate to workspace/dashboard view with project context, not directly to document editor

### P1 Major Issues (2)

6. **Analysis Pane - doesn't fill horizontally**
   - Symptom: Code pane only expands vertically, width is fixed
   - Severity: P1 (UX issue, but code runs)
   - Root Cause: CSS width property hardcoded, flex/grid not configured
   - Fix: Update CSS for full-width expansion

7. **3-Dot Menu - appears on both tiles**
   - Symptom: Menu appears on both recent and all projects tiles when clicked
   - Severity: P1 (UX confusion)
   - Root Cause: Event delegation triggering on all tiles, state not tracking which tile clicked
   - Fix: Scope menu to clicked tile only, prevent event propagation

## Success Criteria

- All 5 P0 issues resolved (literature, chat, bibliography, navigation, project creation)
- All 2 P1 issues resolved (analysis pane, 3-dot menu)
- Achieve >= 90% pass rate on re-test
- No regressions introduced

## Product Decision Required

**Project vs Document Concept:**
The regression testing revealed a fundamental product question:
- Is a "project" a document? (current implementation)
- Or is a "project" a workspace containing multiple documents? (user expectation)

**Recommendation:** For this gap closure plan, implement minimal fix:
- Create project -> navigate to dashboard/workspace view with project context selected
- User can then create documents within that project context
- Defer full workspace-vs-document architecture decision to Phase 23
