# Plan 03-01: Memory Backend Data Model

**Objective:** Design and implement database schema for storing claims, findings, preferences, and their relationships

## Context

Phase 3 requires a memory and information graph backend to store research claims extracted from literature, findings from data analyses, user preferences, and relationships between claims. This is the foundational data layer that all subsequent memory functionality depends on.

### Research Findings

From discovery research on PostgreSQL graph patterns:

**Adjacency List vs Closure Table:**
- **Adjacency List**: Simpler schema, better for frequent updates, requires recursive CTEs for deep traversals
- **Closure Table**: O(n) traversal performance, higher storage, more complex maintenance

**Decision**: Use adjacency list with recursive CTEs because:
1. Claim relationships will be updated frequently as AI extracts new insights
2. Simpler schema reduces migration risk
3. PostgreSQL recursive CTEs perform well for moderate-sized graphs (<100K nodes)
4. Can migrate to closure table later if performance issues arise

**JSONB vs Separate Tables:**
- **JSONB**: Flexible schema, GIN indexes for search, simpler for evolving claim metadata
- **Separate Tables**: Type-safe, better for complex relational queries

**Decision**: Use JSONB for claim/finding metadata because:
1. Claim structure may evolve based on literature domain (medical vs CS vs social sciences)
2. GIN indexes enable efficient full-text search on claim contents
3. Type enforcement can be added at application layer
4. Existing codebase uses JSONB for task_counts, phases with success

**Sources:**
- [Recursive Query vs Closure Table vs Graph Database](https://blog.getdatascale.com/recursive-query-vs-closure-table-vs-graph-database-a-complete-guide-from-my-pov-2a8dd794b733)
- [LTREE vs Adjacency List vs Closure Table](https://dev.to/dowerdev/implementing-hierarchical-data-structures-in-postgresql-ltree-vs-adjacency-list-vs-closure-table-2jpb)
- [PostgreSQL JSONB Performance](https://www.dbvis.com/thetable/json-vs-jsonb-in-postgresql-a-complete-comparison/)
- [Knowledge Graph Schema Design Patterns](https://medium.com/terminusdb/knowledge-graph-schema-design-patterns-and-principles-3d8761f4f37f)

### Task 1: Design Claim model with provenance tracking
**File:** `backend/database/models.py`
**Duration:** 10min

Create the Claim model to store research claims extracted from papers, files, and analyses:

```python
class ClaimSourceType(str, enum.Enum):
    """Types of sources that can produce claims."""
    PAPER = "paper"
    FILE = "file"
    ANALYSIS = "analysis"
    USER = "user"

class Claim(Base):
    """
    Research claim extracted from literature, files, or analyses.
    Claims represent assertions, facts, or findings that can be cited and related.
    """
    __tablename__ = "claims"

    id = Column(String(36), primary_key=True, default=generate_uuid)
    project_id = Column(String(36), ForeignKey("projects.id", ondelete="CASCADE"), nullable=False)

    # Claim content
    claim_text = Column(Text, nullable=False, index=True)
    claim_type = Column(String(50), nullable=True)  # assertion, fact, finding, hypothesis

    # Flexible metadata (e.g., methodology, sample_size, p_value for scientific claims)
    claim_data = Column(JSONB, default=dict)

    # Provenance tracking
    source_type = Column(SQLEnum(ClaimSourceType), nullable=False)
    source_id = Column(String(36), nullable=False)  # ID of paper/file/analysis/user
    confidence = Column(Float, default=0.0)  # 0.0 to 1.0

    # Relevance prioritization
    relevance_score = Column(Float, nullable=True)  # Calculated based on project context

    # Extraction metadata
    extracted_at = Column(DateTime(timezone=True), default=utc_now, nullable=False)
    extracted_by = Column(String(36), nullable=True)  # User or system ID

    # Relationships
    relationships_as_source = relationship("ClaimRelationship", foreign_keys="ClaimRelationship.from_claim_id", back_populates="from_claim")
    relationships_as_target = relationship("ClaimRelationship", foreign_keys="ClaimRelationship.to_claim_id", back_populates="to_claim")

    __table_args__ = (
        Index("idx_claims_project_id", "project_id"),
        Index("idx_claims_source", "source_type", "source_id"),
        Index("idx_claims_relevance", "relevance_score"),  # For prioritization
        Index("idx_claims_claim_text_gin", "claim_text", postgresql_using="gin", postgresql_ops={"claim_text": "gin_trgm_ops"}),  # Full-text search
    )
```

**Success criteria:**
- Claim model created with all columns above
- Foreign key to projects.id with CASCADE delete
- Indexes on project_id, (source_type, source_id), relevance_score
- GIN index on claim_text for full-text search (requires pg_trgm extension)
- Enums imported and defined

### Task 4: Design ClaimRelationship model for graph edges
**File:** `backend/database/models.py`
**Duration:** 8min

Create the ClaimRelationship model to track relationships between claims:

```python
class RelationshipType(str, enum.Enum):
    """Types of relationships between claims."""
    ASSOCIATION = "association"
    CORRELATION = "correlation"
    CAUSALITY = "causality"
    PREREQUISITE = "prerequisite"
    CONTRADICTION = "contradiction"
    SUPPORT = "support"
    ELABORATION = "elaboration"

class ClaimRelationship(Base):
    """
    Relationship between two claims.
    Forms a graph structure enabling traversal of connected claims.
    """
    __tablename__ = "claim_relationships"

    id = Column(String(36), primary_key=True, default=generate_uuid)
    project_id = Column(String(36), ForeignKey("projects.id", ondelete="CASCADE"), nullable=False)

    # Edge endpoints
    from_claim_id = Column(String(36), ForeignKey("claims.id", ondelete="CASCADE"), nullable=False)
    to_claim_id = Column(String(36), ForeignKey("claims.id", ondelete="CASCADE"), nullable=False)

    # Relationship type and metadata
    relationship_type = Column(SQLEnum(RelationshipType), nullable=False)
    strength = Column(Float, default=0.5)  # 0.0 to 1.0, confidence in relationship
    metadata = Column(JSONB, default=dict)  # Additional context (e.g., "via paper X", "contradicts figure Y")

    # Timestamps
    created_at = Column(DateTime(timezone=True), default=utc_now, nullable=False)

    # Relationships
    from_claim = relationship("Claim", foreign_keys=[from_claim_id], back_populates="relationships_as_source")
    to_claim = relationship("Claim", foreign_keys=[to_claim_id], back_populates="relationships_as_target")

    __table_args__ = (
        UniqueConstraint("from_claim_id", "to_claim_id", "relationship_type", name="uq_claim_relationships_edge"),
        Index("idx_claim_relationships_project_id", "project_id"),
        Index("idx_claim_relationships_from_claim", "from_claim_id"),
        Index("idx_claim_relationships_to_claim", "to_claim_id"),
        Index("idx_claim_relationships_type", "relationship_type"),
    )
```

**Success criteria:**
- ClaimRelationship model created with all columns above
- Unique constraint on (from_claim_id, to_claim_id, relationship_type)
- Indexes on project_id, from_claim_id, to_claim_id, relationship_type
- Bidirectional relationships to Claim model

### Task 5: Create migration script for memory tables
**File:** `backend/scripts/migrate_add_memory_tables.py`
**Duration:** 10min

Create migration script that:
1. Creates claims, findings, preferences, claim_relationships tables
2. Enables pg_trgm extension for text search
3. Creates GIN indexes on claim_text and finding_text
4. Adds recursive CTE helper function for graph traversal

```python
#!/usr/bin/env python3
"""Migration: Add memory and information graph tables."""
import asyncio
from sqlalchemy import text
from database.session import get_async_session

async def upgrade():
    """Add memory tables to database."""
    async for session in get_async_session():
        async with session.begin():
            # Enable pg_trgm for text search
            await session.execute(text("CREATE EXTENSION IF NOT EXISTS pg_trgm;"))

            # Create helper function for recursive claim traversal
            await session.execute(text("""
                CREATE OR REPLACE FUNCTION get_related_claims(
                    p_project_id VARCHAR(36),
                    p_claim_id VARCHAR(36),
                    p_max_depth INTEGER DEFAULT 3
                )
                RETURNS TABLE (
                    claim_id VARCHAR(36),
                    claim_text TEXT,
                    relationship_type VARCHAR(50),
                    depth INTEGER,
                    path VARCHAR(255)[]
                ) AS $$
                DECLARE
                BEGIN
                    RETURN QUERY
                    WITH RECURSIVE claim_graph AS (
                        -- Base case: direct relationships
                        SELECT
                            c.id,
                            c.claim_text,
                            cr.relationship_type,
                            1 as depth,
                            ARRAY[c.id] as path
                        FROM claims c
                        JOIN claim_relationships cr ON cr.to_claim_id = c.id
                        WHERE cr.from_claim_id = p_claim_id
                          AND c.project_id = p_project_id

                        UNION ALL

                        -- Recursive case: follow relationships
                        SELECT
                            c.id,
                            c.claim_text,
                            cr.relationship_type,
                            cg.depth + 1,
                            cg.path || c.id
                        FROM claims c
                        JOIN claim_relationships cr ON cr.to_claim_id = c.id
                        JOIN claim_graph cg ON cg.claim_id = cr.from_claim_id
                        WHERE c.project_id = p_project_id
                          AND cg.depth < p_max_depth
                          AND NOT c.id = ANY(cg.path)  -- Prevent cycles
                    )
                    SELECT * FROM claim_graph
                    ORDER BY depth, relationship_type;
                END;
                $$ LANGUAGE plpgsql;
            """))

            print("✓ Migration complete: Memory tables created")
            print("  - claims table with provenance tracking")
            print("  - findings table for analysis results")
            print("  - preferences table for user settings")
            print("  - claim_relationships table (adjacency list)")
            print("  - pg_trgm extension enabled for text search")
            print("  - get_related_claims() function for graph traversal")

if __name__ == "__main__":
    asyncio.run(upgrade())
```
